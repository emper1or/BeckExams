
- что такое ORM
- рассказать про 2 паттерна, 2 подхода

ORM (Object–Relational Mapping) - подход который позволяет работать с БД при помощи классов их методов и объектов классов, а не SQL запросов.

Паттерны:
1.  Data Mapper
	   объект ничего не знает о сущестовании бд, за запись данных отвечает отдельный слой - репозиторий
	   Хорошо, что логика вынесена отдельно от моделей, можно тестировать бл отдельно (круто SQL Alchymy) 
2. Active Record
	   подразумевает, что одна строка - один объект класса, он имеет логику работы с бд, может обновляться, добавляться просто.  Благодаря этому мы получаем меньше кода, но сложно масштабировать и сильная связь между моделью и логикой (Django orm)

 Подходы:
 Code First: определяются модели в коде, затем генерируете схема базы данных.

Database First: создается схема базы данных, затем генерируются модели.

Мы писали Data Mapper + Code First
