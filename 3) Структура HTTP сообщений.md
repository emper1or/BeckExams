## 3) Структура HTTP сообщений
Структура HTTP сообщений (заголовки, тело запроса и т.д):
- Методы. Разница между PUT vs PATCH.
- Коды: 200, 201, 204, 403, 404
- Безопасные методы и не безопасные, идемпотентность
- Что такое Accept и Content Type. С помощью какого Content Type можно загружать файлы на сервер

## 3.1 Структура HTTP-запроса

`METHOD /path HTTP/1.1 Headers...  Body`

### Части:

- стартовая строка
    
- заголовки
    
- тело (необязательно)
    

---

## 3.2 HTTP методы

### Основные методы

- `GET` — получить данные
    
- `POST` — создать ресурс
    
- `PUT` — полностью заменить
    
- `PATCH` — частично изменить
    
- `DELETE` — удалить
    

---

### PUT vs PATCH

|PUT|PATCH|
|---|---|
|заменяет ресурс целиком|обновляет часть|
|идемпотентен|может быть не идемпотентен|

---

## 3.3 HTTP коды ответа

- `200 OK` — успешный запрос
    
- `201 Created` — ресурс создан
    
- `204 No Content` — успешно, без тела
    
- `403 Forbidden` — доступ запрещён
    
- `404 Not Found` — ресурс не найден
    

---

## 3.4 Безопасные и идемпотентные методы

### Безопасные

- не изменяют состояние сервера
    
- `GET`, `HEAD`
    

### Идемпотентные

- повтор запроса не меняет результат
    
- `GET`, `PUT`, `DELETE`
    

---

## 3.5 Accept и Content-Type

- `Content-Type` — **формат отправляемых данных**
    
- `Accept` — **какой формат ожидает клиент**
    

Примеры:

`Content-Type: application/json Accept: application/json`

### Загрузка файлов

`Content-Type: multipart/form-data`
