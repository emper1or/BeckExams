
Зачем версионировать API. Способы версионирования.


Версионирования API для добавления новых функций без нарушения работы старых клиентов.

**Версионирование:**  
Позволяет вносить изменения в API без нарушения работы существующих клиентов.  
Как делают:  
URI версионирование: включение номера версии в URL (например, /api/v1/book, /api/v2/book).  
Версионирование через заголовки: использование HTTP заголовков для указания версии API.  Accept-Version: 2
Версионирование через параметры запроса: добавление версии как параметра запроса. GET /api/items?version=1
